Pagina 04

Arquitetura tecnica e infraestrutura necessaria

O sistema opera hoje como um monolito modular com FastAPI, frontend estatico e banco PostgreSQL, publicado por proxy reverso e empacotado com Docker Compose.

Arquitetura logica

Como as camadas se organizam

Frontend

HTML, CSS e JavaScript vanilla servidos pela propria aplicacao.

Backend

FastAPI com routers, services, repositories, models, schemas e core.

Banco

PostgreSQL relacional com migrations Alembic, historicos e tabelas de governanca.

Uploads

Arquivos PDF gravados fora da raiz publica, com metadados e controle de acesso.

Proxy

Nginx ou Nginx Proxy Manager como camada de HTTPS e exposicao controlada.

Seguranca

JWT, RBAC, rate limit de login, docs protegidos e endurecimento para producao.

Infra minima

O que e necessario para subir em outro ambiente

Servidor LinuxCom Docker, Docker Compose e acesso administrativo controlado.
Container webAplicacao FastAPI servindo API e frontend estatico.
Container dbPostgreSQL com volume persistente e backup regular.
Proxy reversoNginx ou NPM para 80/443, redirect para HTTPS e certificado valido.
DNSDominio apontando para o proxy publico ou rede publicada.
StorageDiretorio persistente para uploads, preferencialmente com backup separado.
SegredosJWT secret, credenciais do banco e admin inicial fora do repositorio.
ObservabilidadeHealth check, logs, teste de restore e rotina minima de verificacao.
